CSV Setup Instructions

The import is strict enough to protect the data, but flexible enough to accept common spreadsheet headers. Use the preferred headers when possible. Extra custom columns are allowed and ignored for now, so outside spreadsheets do not need to be stripped down.

Judging Classes CSV
Preferred header row:
class_code,class_name,description,year_min,year_max,make_contains,model_contains,keyword_contains,award_places,sort_order,is_active
Accepted aliases include: code, class, class name, judging class, min year, max year, make, model, keyword, awards, places, active.
Accepted Registrations CSV
Preferred header row:
car_number,owner_name,phone,email,year,make,model,class_code,class_name,payment_status,waiver_received,custom_1,custom_2
Accepted aliases include: name, full name, owner, mobile, cell, email address, vehicle year, vehicle make, vehicle model, car #, entry number, class, division, judging class.
Important: Registration imports will create missing judging classes when a row includes class_code, class_name, class, division, or judging_class that does not already exist for this show.

Import Judging Classes

Use this first when you have a class list. Existing classes with the same class code or class name are updated.

Import Accepted Car Registrations

Use this for cars already accepted outside the platform. If car number is blank, the next open placeholder card is used; if none exists, the next available number is used.

Imported registrations are assumed already accepted through the outside system. They will not be sent to online checkout.

Google Forms / Spreadsheet Export

In Google Forms or Google Sheets, use File → Download → Comma Separated Values (.csv), then upload here. You may leave one or two extra custom columns in the file; they will be ignored unless mapped in a future release.
